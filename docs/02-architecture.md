## 2. 核心业务流

### 2.1 文档入库流程
```
上传文件 → 格式检测 → 异步解析 → 文本提取 → 公式识别(OCR) → 
文本分块(Recursive) → Embedding(BGE) → 存入 ChromaDB → 元数据入库
```

### 2.2 AI 备课流程
```
用户输入: "准备牛顿第二定律的教案"
    ↓
检索: 相关教材章节 + 典型例题 + 常见误区
    ↓
LLM 生成: 教学目标 + 教学流程 + 推荐实验/视频
    ↓
工具调用: 生成示意图代码(matplotlib/TikZ)
    ↓
输出: 完整教案 + 可编辑图表代码 + 视频链接
```

### 2.3 AI 答疑流程
```
用户输入: 学生问题(文字/图片)
    ↓
如果是图片 → OCR 识别 + 公式提取
    ↓
检索: 相似题目 + 知识点 + 解题方法
    ↓
LLM 推理: 分步解析 + 物理原理说明 + LaTeX 公式
    ↓
输出: 结构化答案(步骤 + 公式 + 图示建议)
```

### 2.4 AI 出卷流程
```
用户输入: 范围(必修一第3章)、难度(中等)、题型(选择4+计算2)
    ↓
检索: 题库中符合条件的题目
    ↓
LLM: 根据要求组合/生成题目
    ↓
工具调用: 生成变式题 + 计算答案
    ↓
输出: LaTeX 格式的完整试卷 + 答案解析
```

## 3. 核心模块设计

### 3.1 模块职责

| 模块 | 职责 | 关键类/函数 |
|------|------|-------------|
| `document_processor` | 多格式文档解析 | `DocumentLoader`, `OCRProcessor` |
| `text_splitter` | 智能分块策略 | `RecursiveTextSplitter`, `SemanticSplitter` |
| `embedding` | 文本向量化 | `BGEEmbedding`, `EmbeddingCache` |
| `vector_store` | 向量存储与检索 | `ChromaStore`, `HybridRetriever` |
| `rag_chain` | RAG 核心链路 | `RetrievalQAChain`, `ContextualCompressor` |
| `agent` | AI Agent 执行 | `TeachingAgent`, `ToolExecutor` |
| `tools` | 工具定义与实现 | `ChartTool`, `VideoTool`, `LaTeXTool` |
| `frontend` | 用户界面 | React 组件、API Client |
